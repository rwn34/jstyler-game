📋 UPDATE LOG

v1.1.14 — Bug fixes + Consumable persistence + Resurrect nerf
  • Tablet rank chip: stats card height 52→68px, comfortable padding, line-height 1.4 — text no longer cropped by border.
  • Daily Chest button shows ✅ icon (instead of 🎁) when claimed, half opacity, click disabled.
  • Daily Master Chip now picks from levels you've UNLOCKED (was picking from all 20, sometimes locked levels). New players see today's daily on a level they can actually play.
  • "DAILY MASTER" → "DAILY GEM" so it's clearer this is for everyone, not master-only. Champions still get +5 gold, others get +5 silver.
  • Stage select dot navigation: max-width clamped to thumbnail width (min(90vw, 320px)), dots reordered to flow under the global stats strip via CSS order.
  • ⚡ Resurrect cooldown 5min → 2min — much more usable.
  • Consumables persist across retries. Death no longer consumes inventory. Shield with 1 hit remaining stays at 1 hit on retry. Triple Jump and Time Freeze stay active. Only WIN or EXIT TO STAGE SELECTION consumes the inventory item.
  • Triple Jump activation now consistent: deferred decrement (was eagerly decremented on activate). Now matches dblshield/timefreeze pattern.
  • Shopkeeper tip in store now shows "1 / 50 ▶" with arrow indicator so it's obvious you can tap to cycle.
v1.1.13 — Auto-retry single setting
  • Auto-retry consolidated to one setting "Auto-retry on death" with options: No auto / Instant / 0.5s / 1s / 2s. Death screen always appears; the chosen value is how long to wait before auto-retrying.
  • Settings migration: existing v1.1.9 setups (autoRetry on/off + deathScreenDelay) auto-converted on load — if previously enabled with 1s delay, becomes "1s" auto-retry; if previously off, becomes "No auto".
  • Tap-to-retry still works at any time after a 300ms safety gate (prevents stray taps from in-game).
v1.1.12 — Polish: retry placement, rank chip, daily indicators
  • In-game pause: 🔄 RETRY LEVEL button moved to the bottom action group between APPLY & RESUME and BACK TO STAGE SELECTION (was floating above QUICK SHOP).
  • Stage select rank chip: text no longer cropped/clipped — bumped padding 1px → 3px vertical, font 0.5 → 0.55rem, explicit line-height 1.3.
  • Daily Chest button on stage select: when claimed, stays visible at half opacity (no longer disappears). Click is disabled while claimed.
  • 💎 Daily Master Chip indicator added: dot in stage navigation now shows a 💎 marker for today's daily stage (or ✓ if already claimed). Stage info tags also display "💎 DAILY MASTER" / "💎 DAILY (claimed)".
v1.1.10 — Reflex Dash Toggle
  • Champions can now toggle Reflex Dash ENABLE/DISABLE from the store. Defaults to ENABLED on first unlock.
  • When you equip Air Dash → Reflex Dash auto-disables (AD covers the basic case; having both was redundant).
  • When you sell Air Dash → Reflex Dash auto-disables too (clean slate). Re-enable manually if you want it back.
  • Manually enabling Reflex Dash → unequips Air Dash automatically (mutual exclusion).
  • While disabled, Reflex Dash grants no Air Dash effect either — you have to equip a separate Air Dash if you want any dash.
v1.1.9 — Auto-retry & Pause Retry
  • 🔄 RETRY LEVEL button added to in-game pause panel — restart the current level on demand without dying first or backing out.
  • Auto-retry on death setting (default OFF). When enabled, after the configurable delay, the level auto-restarts without a tap.
  • Death screen delay setting: 0.5s / 1s / 2s (default) / 3s. Controls how long the death banner stays visible.
  • Both settings available in stage-select Settings AND in-game pause panel — toggle anywhere, persists between sessions.
  • Tap-to-retry still works as fallback when auto-retry is on (instant retry before timer fires).
v1.1.8 — Gold Economy Fix
  • Master gems now actually give +2 gold each (champion-only). Previously the credit was silently absorbed by an anti-cheat clamp — your master gem collections gave nothing.
  • Daily Champion bonus now gives +5 gold as advertised. Same fix.
  • Retroactive credit: every master gem you've already collected is automatically added to your balance on next load (≈ 2 gold per master gem).
  • Gem ratio display fixed. Previously stored gem-collection state at sparse indices, then `normalizeBestChips` truncated indices > 6, silently losing 4 of every 6 collections on each page reload. Now stored as packed (0..maxGems-1).
  • Migration: existing gem-collection data is auto-packed on next load. Caps at level's max gems (e.g. 7/7) — players who legitimately cleared a level keep full credit.
  • Buy/sell skills now uses straightforward arithmetic — no more anti-cheat clamp eating gold.
  • Self-heal: if `goldSpent` went negative from old bug, auto-resets to current owned skills cost on next load.
v1.1.7 — Telemetry Naming Fix
  • Session start now deferred until a player name is set (returning users send immediately, new users send after onboarding)
  • Heartbeats only fire after the player has named themselves (no nameless online-counting)
  • New 'name_set' event fires when a player picks/changes their name — fuels the metrics funnel
  • No gameplay impact — pure metrics quality fix
